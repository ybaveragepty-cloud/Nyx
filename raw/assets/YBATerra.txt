Agent operating brief

The agent rules established were:

Build only from the numbered task list.
Complete dependencies before higher layers.
Every task must output code, tests, migrations, and notes.
No invented pages or tables outside the approved model.
Reuse shared entities and UI primitives.
All write actions must be permission-checked.
All critical actions must be audit-logged.
Prefer modular, replaceable architecture over shortcuts.
Mark blockers explicitly rather than guessing.
Update task state using: not started, in progress, blocked, done.
Critical dependency rules

The following were declared non-negotiable:

Do not build feature screens before the entity model exists.
Do not build task execution before permissions exist.
Do not build dashboards before KPI definitions exist.
Do not build configurable workflows before workflow versioning exists.
Do not build production modules without batch/lot traceability.
Do not build alerts without an incident audit trail.
Do not connect AI/agents directly to production write actions without permission gates and logs.

Every record belongs to an organization.
Most operational records link to a site and often a specific location node.
Anything produced, moved, treated, harvested, packed, or sold should have a batch/lot/cohort identity.
Every action should be traceable to who, when, where, and what.
No workflow should exist without linking to a real business object.
Configuration must be tenant-aware.

Global schema rules
Most tables should include:
id UUID PK
organization_id where applicable
created_at
updated_at
Use UUID PKs on all business tables.
Do not soft delete everything; prefer business status.
Never hard delete traceability or audit records.
All tenant-owned records must include organization_id.
Operational records should include site_id.
Work/measurements/assets/production should include location_node_id where meaningful.
Human-readable codes must be unique within correct scope.
Writes to critical entities must create audit log events.
Use DB enums only for highly stable values.
Use reference tables where values may expand.
